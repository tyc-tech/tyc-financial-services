# 竞品对比分析

## 触发条件

行业研究、目标企业护城河评估、并购对象筛选、行业地图绘制。

关键词：竞品、行业格局、horizontal compare、护城河、行业地图

## 输入要求

- **searchKey** (必填): 主标的企业名称 或 信用代码
- **competitors** (可选): 自定义竞品名称数组（若不提供则自动从 `get_competitors` 拉取）

## 执行流程

### Step 1: 主标的基本面
- `get_company_registration_info`
- `get_company_scale`

### Step 2: 自动识别竞品（若未指定 competitors）
- `get_competitors` — 拉取系统识别的竞品列表

### Step 3: 对每个竞品（main + competitors）批量获取
- `get_company_registration_info`
- `get_patent_info` — 专利总数
- `get_bidding_info` — 中标金额/项次
- `get_qualifications` — 资质证书数量
- `get_financing_records` — 融资轮次

### Step 4: 横向对比
聚合上述数据生成对照表

## 输出格式

```markdown

## 一、对比对象
| 序号 | 企业 | 关系 |
|------|------|------|
| 1 | {主标的 name} | 主标的 |
| 2-N | ... | 自动识别竞品 |

## 二、横向对比矩阵
| 指标 | 主标的 | 竞品 1 | 竞品 2 | ... |
|------|--------|--------|--------|-----|
| 注册资本 | ... | ... | ... | ... |
| 参保人数 | ... | ... | ... | ... |
| 专利数 | ... | ... | ... | ... |
| 中标项次 | ... | ... | ... | ... |
| 资质证书 | ... | ... | ... | ... |
| 融资总轮次 | ... | ... | ... | ... |
| 综合排名 | 1 | 2 | 3 | ... |

## 三、护城河深度评估
- 知产壁垒: 主标的 vs 行业均值
- 资质壁垒: ...
- 资本壁垒: ...

## 四、投资建议
- 主标的行业地位: 龙头 / 第二梯队 / 跟随者
- 主要风险: 同质化 / 替代品 / ...
```
